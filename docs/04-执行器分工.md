## 4.1 三类执行器

```
┌──────────────────────────────────────────────────────────────┐
│  任务工单 (job) — 唯一出口，所有执行器 claim 后才能跑         │
│  规则: 同一 (stage, fingerprint) 只能有一个 running job     │
└──────────────────────────────────────────────────────────────┘
```

### 执行器总览

| 执行器 | 定位 | 适用场景 |
|--------|------|---------|
| **Codex** | 主推进器 | JS逆向、算法提取、脚本生成、主流任务 |
| **M2.7** | 二验/辅助 | second-opinion、文档、审查、Codex备选 |
| **专用工具** | 纯执行 | mitmproxy、Frida、Chrome、Jadx、APKTool |

## 4.2 Codex 执行器

> 通过 `codex-task-exec.sh` 调用 Claude Code 执行逆向任务

### 适用任务

```
✓ Web JS 逆向（签名算法提取）
✓ 混淆 JS 还原
✓ API 协议分析
✓ Python 脚本生成
✓ APP Java 层分析
✓ APP 签名算法定位
✓ APP Python 实现
✓ 交付脚本撰写
```

### 使用方式

```bash
# 手动触发
bash scripts/codex-task-exec.sh \
    --task-id "web-sig-taobao-20260422" \
    --stage "reverse" \
    --prompt-file "prompts/reverse-web-sig.md" \
    --context "tasks/status/web-sig-taobao/latest.json"
```

### 优势

- 长上下文 (200K token)，适合分析大文件 JS
- 代码生成能力强
- 擅长处理复杂混淆逻辑

## 4.3 M2.7 执行器

> 通过 Hermes Agent 直接执行，或通过 `opencode-task-exec.sh`

### 适用任务

```
✓ second_opinion — Codex 结果复核（最重要）
✓ doc_synthesis — 生成逆向文档（README、接口文档）
✓ alt_analysis — 方案对比（Codex 和 M2.7 各出一版，对比选优）
✓ sanity_check — 脚本健全性检查（语法、边界、异常处理）
✓ quick_fix — 小型 blocker 修复（不需要重跑全链路）
✓ app_web_concat — Web + APP 联合验证
```

### 使用方式

```bash
# 触发二验
python3 scripts/auto-task-runner.py second-opinion \
    --source-result "tasks/status/web-sig-taobao/step-07-verify/latest.json"
```

## 4.6 claim 机制（防重）

```python
# 每个工单执行前必须 claim

def claim_job(job):
    lock_file = f"tasks/status/{job.task_id}/{job.stage}.lock"
    if os.path.exists(lock_file):
        raise JobAlreadyClaimedError(job.job_id)

    with open(lock_file, "w") as f:
        f.write(json.dumps({
            "job_id": job.job_id,
            "executor": os.environ.get("EXECUTOR_TYPE", "unknown"),
            "claimed_at": datetime.utcnow().isoformat()
        }))

    job.status = "running"
    job.executor_type = os.environ.get("EXECUTOR_TYPE")
    save_job(job)

def release_job(job, status, result):
    # 删除 lock 文件
    lock_file = f"tasks/status/{job.task_id}/{job.stage}.lock"
    if os.path.exists(lock_file):
        os.remove(lock_file)

    job.status = status
    job.finished_at = datetime.utcnow().isoformat()
    job.result_paths = result.paths
    save_job(job)
```

## 4.7 双执行器协同示例

```
场景: 某网站 sign 参数逆向

Step 1: Codex 主逆向
  → claim job "web-reverse-001"
  → 分析 JS 代码
  → 生成 taobao-signature.py
  → verify: 10/10 命中

Step 2: M2.7 二验
  → claim job "web-second-opinion-001"
  → 读取 Codex 结果
  → 审查代码质量
  → 发现一个问题：时间戳格式少处理了毫秒
  → 输出 review-report.md

Step 3: Codex 修复
  → 收到 review
  → 修复时间戳问题
  → verify: 10/10 命中

Step 4: 归档
  → 所有结果写入 memory 层
  → Git 提交
  → 向量导入 LanceDB
```
