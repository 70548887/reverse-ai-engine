## 9.2 Git 提交规范

### 提交信息格式

```
<类型>(<范围>): <简短描述>

[可选的详细正文]

[可选的脚注]
```

### 类型

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | 修复 bug |
| docs | 文档更新 |
| style | 格式调整（不影响代码） |
| refactor | 重构（不影响功能） |
| perf | 性能优化 |
| test | 测试相关 |
| chore | 构建/工具变更 |

### 示例

```bash
git commit -m "feat(web): 新增 AES-256-CBC 签名提取流程

- 新增 browser-control.py 的 hook_console_crypto 方法
- 更新 reverse-controller.py 支持 AES 验证
- 添加 5 个测试用例验证

Closes: #12
"
```

## 9.3 分支管理

```
main          ← 稳定版，始终可部署
├── develop   ← 开发版，功能集成后合并
│   ├── feature/web-sign-aes      ← 功能分支
│   ├── feature/app-frida-hook
│   └── fix/debugger-bypass
└── hotfix/*  ← 紧急修复，从 main 切出
```

### 工作流程

```bash
# 1. 从 develop 创建功能分支
git checkout develop
git pull
git checkout -b feature/web-sign-extraction

# 2. 开发 + 提交
git add .
git commit -m "feat: ..."

# 3. 推送 + 创建 PR
git push -u origin feature/web-sign-extraction
gh pr create --base develop --head feature/web-sign-extraction

# 4. Code Review → 合并 → 删除分支
```

## 9.7 测试规范

```python
# tests/test_signature.py

def test_aes_signature():
    """AES-256-CBC 签名测试"""
    from signature import make_sign

    result = make_sign({"keyword": "手机"})
    assert result == "abc123..."

def test_timestamp_tolerance():
    """时间戳容差测试（允许 ±50ms）"""
    from signature import make_sign_with_time

    ts = int(time.time() * 1000)
    sig = make_sign_with_time({"keyword": "手机"}, ts)
    # 模拟 30ms 后的请求
    ts2 = ts + 30
    sig2 = make_sign_with_time({"keyword": "手机"}, ts2)
    assert sig == sig2  # 时间戳不同但应该在容差内

def test_invalid_params():
    """错误参数测试"""
    from signature import make_sign

    with pytest.raises(ValueError):
        make_sign("not a dict")
```

## 9.8 错误日志规范

```python
# 使用 loguru，记录上下文

logger.info("开始逆向任务", task_id=task_id, target=target_url)
logger.debug("抓包完成", request_count=len(requests))
logger.warning("检测到反调试", url=url)
logger.error("签名验证失败", sample=sample_id, expected=expected, got=got)
logger.success("任务完成", task_id=task_id, pass_rate=pass_rate)
```

日志级别:
- **DEBUG**: 详细调试信息
- **INFO**: 正常流程节点
- **SUCCESS**: 关键步骤成功
- **WARNING**: 异常但可处理
- **ERROR**: 失败，需要关注
