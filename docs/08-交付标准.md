## 8.1 交付物定义

每个逆向任务完成后，必须产出以下交付物：

```
逆向任务交付物
├── 签名脚本（必须）
│   └── <target>-signature.py       # 可运行的 Python 签名脚本
├── 逆向文档（必须）
│   └── <target>-report.md          # 逆向分析报告
├── 验证结果（必须）
│   └── verify-result.json          # 验证数据（通过/失败率）
├── 中间产物（可选）
│   ├── capture.har                # 抓包原始数据
│   ├── deobfuscated.js             # 还原后的 JS
│   └── frida-log.txt               # Frida Hook 日志
└── 元数据（必须）
    └── task-meta.json              # 任务元数据 + SHA256
```

## 8.2 签名脚本标准

### 必须满足

1. **独立运行** — 不依赖逆向工具（mitmproxy/Frida 等），纯 Python 即可跑
2. **可复现** — 相同输入产生相同输出
3. **带注释** — 关键算法逻辑必须有中文注释
4. **有测试用例** — 内置至少 3 个测试用例验证
5. **版本号** — 文件头部注明支持的 API 版本

## 8.5 结果归档标准

### 每轮执行必须归档

```python
# archive_result.py

def archive(task_id, stage, result):
    output_dir = Path(f"memory/projects/{task_type}/{task_id}")

    # 1. result.json
    (output_dir / f"{stage}-result.json").write_text(
        json.dumps(result, indent=2, default=str)
    )

    # 2. log.txt
    if result.log:
        (output_dir / f"{stage}-log.txt").write_text(result.log)

    # 3. SHA256
    sha256_map = {}
    for key, path in result.files.items():
        sha256_map[key] = hash_file(path)

    # 4. memory-sync.json
    (output_dir / "memory-sync.json").write_text(json.dumps({
        "task_id": task_id,
        "stage": stage,
        "archived_at": datetime.utcnow().isoformat(),
        "sha256": sha256_map
    }, indent=2))

    # 5. 镜像到各层
    mirror_to_daily(task_id, stage, result)
    mirror_to_project(task_id, stage, result)
    mirror_to_raw(task_id, stage, result)
    mirror_to_history(task_id, stage, result)
```
