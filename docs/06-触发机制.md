## 6.1 四种触发方式

```
┌─────────────────────────────────────────────────────────────┐
│  触发优先级（从上到下，0最高优先）                           │
├────┬────────────────────────┬───────────────────────────────┤
│ 0  │ 人工下发               │ 最高优先，用户/Telegram/CLI   │
│ 1  │ 事件驱动               │ 样本落地即触发，实时响应       │
│ 2  │ API 接收               │ 第三方系统 webhook             │
│ 3  │ 定时自动发现           │ 保底，每6小时扫目标列表       │
└────┴────────────────────────┴───────────────────────────────┘
```

## 6.2 人工下发

### CLI 下发

```bash
# Web 逆向任务
python3 scripts/reverse-controller.py web \
    --target https://example.com/api/search \
    --method POST \
    --output ./results/example-signature

# APP 逆向任务
python3 scripts/reverse-controller.py app \
    --package com.example.app \
    --apk ./tools/apk/example.apk \
    --output ./results/example-app-signature

# 查看任务状态
python3 scripts/reverse-controller.py status --task-id <task_id>
```

## 6.3 事件驱动（主要触发）

### 核心思想

```
旧模式（不好）:  LLM 驱动高频轮询 → 浪费资源，空转多
新模式（好）:    样本落地 → systemd path 触发 → 控制器接手
                        ↓
             timer 只做保底（每30分钟扫一次）
```

### systemd path 单元

```ini
# /etc/systemd/system/reverse-brain.service
[Unit]
Description=Reverse Engineering Brain Controller
After=network.target

[Service]
Type=simple
User=ctyun
WorkingDirectory=/home/ctyun/.openclaw/workspace
ExecStart=/usr/bin/python3 /home/ctyun/.openclaw/workspace/scripts/reverse-controller.py
Restart=on-failure
RestartSec=30
Environment="PYTHONPATH=/home/ctyun/.openclaw/workspace"

# /etc/systemd/system/reverse-intake.path
[Path]
PathChanged=/home/ctyun/.openclaw/workspace/tasks/intake/pending
Unit=reverse-intake.service

[Install]
WantedBy=multi-user.target

# /etc/systemd/system/reverse-intake.service
[Unit]
Description=Reverse Intake Worker
After=reverse-brain.service

[Service]
Type=simple
User=ctyun
WorkingDirectory=/home/ctyun/.openclaw/workspace
ExecStart=/usr/bin/python3 /home/ctyun/.openclaw/workspace/scripts/auto-task-runner.py intake
```

### 触发流程

```
1. 任务文件写入 tasks/intake/pending/<task_id>.jsonl
2. systemd path 检测到文件变化
3. reverse-intake.service 被触发
4. auto-task-runner.py intake 扫描 pending 目录
5. reverse-controller.py 创建工单
6. 控制器决策后分配执行器
```

## 6.4 定时自动发现

### 定时器单元

```ini
# /etc/systemd/system/reverse-sweep.timer
[Timer]
OnBootSec=5min
OnUnitActiveSec=6h      # 每6小时保底扫一次
Persistent=true

[Install]
WantedBy=timers.target

# /etc/systemd/system/reverse-sweep.service
[Unit]
Description=Reverse Engineering Sweep
Requires=reverse-brain.service

[Service]
Type=oneshot
User=ctyun
WorkingDirectory=/home/ctyun/.openclaw/workspace
ExecStart=/usr/bin/python3 /home/ctyun/.openclaw/workspace/scripts/reverse-controller.py sweep
```

### 扫描逻辑

```python
def sweep():
    """定时扫目标列表，检查是否有新版本/新API"""
    targets = read_target_list("config/targets.yaml")

    for target in targets:
        current_version = check_current_version(target)
        if current_version > target.last_known_version:
            # 发现新版本 → 自动创建逆向任务
            create_task(target, reason="new_version")
            update_target_version(target, current_version)

        # 检查 blocked 任务是否可续跑
        for task in get_blocked_tasks(target.name):
            if has_new_sample(task) or has_new_approach(task):
                resume_task(task)
```

## 6.5 API 触发

```python
# scripts/reverse-api-server.py
from flask import Flask, request, jsonify
import os

app = Flask(__name__)

@app.route("/intake", methods=["POST"])
def intake():
    """第三方系统 webhook 触发"""
    task_data = request.json
    task_id = write_to_intake(task_data)

    # 立即触发（不走定时）
    trigger_intake()
    return jsonify({"task_id": task_id, "status": "queued"})

# 启动: python3 scripts/reverse-api-server.py --port 8080
```

### API 请求格式

```bash
curl -X POST http://localhost:8080/intake \
  -H "Content-Type: application/json" \
  -d '{
    "type": "web.js.signature",
    "priority": "high",
    "target": {
      "url": "https://example.com/api/search",
      "method": "POST"
    }
  }'
```

## 6.6 启停管理

```bash
# 查看所有逆向相关单元
systemctl list-units 'reverse-*' 're-sweep*' --all

# 启动（事件驱动模式）
bash scripts/bilibili-controller-start.sh
# 等价于手动:

systemctl enable --now reverse-sweep.timer

# 停止（彻底静止）

systemctl disable --now reverse-intake.path reverse-intake.service
systemctl disable --now reverse-sweep.timer reverse-sweep.service

# 查看日志
journalctl -u reverse-brain -f
```

## 6.7 触发优先级示例

```
场景: 某目标站点刚刚发布了新版本API

事件驱动响应（毫秒级）:
  用户发现 → 通过 Telegram 告知 → 任务写入 intake/pending
  → path 触发 → 30秒内控制器开始决策
  → 执行器开始抓包

定时发现（6小时后）:
  sweep timer 触发 → 发现版本变化
  → 自动创建新任务 → 排队等待执行

结论: 人工/事件驱动 用于"立即要"的任务
       定时发现 用于"不着急但持续跟踪"的任务
```
